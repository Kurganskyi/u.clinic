# Сводный отчет по документу "Напоминания и рассылки для клиники Uclinic"

## Общая информация

**Дата создания:** 30 октября 2025  
**Назначение:** Собрать требования для настройки автоматических напоминаний и рассылок  
**Основная проблема:** WhatsApp блокировался, нужны альтернативные надежные каналы коммуникации

---

## Ключевые проблемные зоны

### 1. Каналы коммуникации
- **Текущая ситуация:** WhatsApp Business с риском блокировок
- **Решение:** Необходимо определить приоритетный канал (Telegram/SMS/Email)
- **Рекомендация:** Telegram в качестве основного канала

### 2. Согласие на рассылки
- **Проблема:** Необходимо явное согласие пациентов на уведомления
- **Вопрос:** Где и как хранятся согласия
- **Критичность:** Высокая (юридические требования)

### 3. Типы сообщений
- Напоминания о записи (за 1 день, за 3 часа)
- Подтверждения записи
- Сообщения после процедуры
- Реактивация пациентов
- Рекламные рассылки

### 4. Интеграции
- Источник данных о записях (CRM/таблица/программа)
- Частота обновления
- Статистика доставки

---

## Варианты решения

### Вариант C — "Быстрый старт" (MVP)
**Стоимость:** 120–220 тыс. ₽  
**Срок:** 1.5–3 недели  
**Включает:**
- Telegram-напоминания (t-24ч, t-3ч)
- Кнопки "Приду"/"Перенести"/"Отменить"
- Простая таблица расписания
- Базовая статистика

**Минусы:**
- Только Telegram
- Нет SMS/Email
- Нет автоматической CRM-синхронизации
- Нет рассылок акций

### Вариант A — "Стандарт"
**Стоимость:** 240–480 тыс. ₽ + SMS ~1–3 ₽/шт  
**Срок:** 3–5 недель  
**Включает:**
- Все из варианта C
- SMS для пациентов без Telegram
- Автоинтеграция с CRM
- Отписки и черные списки
- Подробная статистика

**Минусы:**
- Нет WhatsApp Business API
- Нет Email-рассылок
- Нет панели управления

### Вариант B — "Омниканал"
**Стоимость:** 320–660 тыс. ₽ + SMS + WhatsApp  
**Срок:** 4–7 недель  
**Включает:**
- Все из варианта A
- WhatsApp Business API (официальный)
- Email-рассылки
- Панель управления
- A/B тесты

---

## Юридические требования

### Согласие на обработку ПДн
- Необходимо явное согласие пациента
- Хранение согласий (папка/CRM/таблица)
- Срок хранения данных (рекомендуется 3 года)

### Медицинские предупреждения
- Дисклеймеры о противопоказаниях
- Возрастные ограничения (18+)
- Согласование текстов с юристом/главврачом

---

## Технические требования

### Ограничения частоты сообщений
- Максимум сообщений в день (рекомендуется не более 3)
- Запрещенные часы отправки (рекомендуется 22:00–9:00)

### Возможность отписки
- Команда в боте (/stop)
- Ссылка в SMS/Email
- Через администратора

### Обработка недоставок
- Повтор через 5/10/30 минут (Telegram)
- Резервный канал (SMS/Email)
- Эскалация к администратору

---

## Статистика и аналитика

### Требуемые метрики
- Количество отправленных сообщений
- Процент доставки
- Процент прочтения
- Процент подтверждения визитов
- Конверсия по каналам (Telegram/SMS/Email/WhatsApp)

---

## Рекомендации

### Для быстрого старта
✅ **Выбрать Вариант C** (Telegram only)
- Минимальный бюджет
- Быстрый запуск (2–3 недели)
- Тестирование на реальных пациентах
- Возможность апгрейда до варианта A

### Для максимальной эффективности
✅ **Выбрать Вариант A** (Telegram + SMS)
- Надежность и автоматизация
- Охват всех пациентов
- Отсутствие рисков блокировок
- Разумная стоимость

### Для максимального охвата
✅ **Выбрать Вариант B** (Омниканал)
- Все каналы коммуникации
- Легальный WhatsApp Business API
- Маркетинговые возможности
- Панель управления

---

## Ежемесячная поддержка

**Стоимость:** 15–40 тыс. ₽/мес + расходы на SMS/WhatsApp

**Включает:**
- Техподдержка (пн–пт, 10:00–19:00)
- Правки текстов
- Запуск рассылок
- Мониторинг доставки
- Ежемесячные отчеты
